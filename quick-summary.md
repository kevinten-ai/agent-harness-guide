# Agent Harness 核心理念速览

## 一句话版本

> **LLM 是大脑，Harness 是身体。** 大脑再聪明，没有手脚（工具）、没有记忆（状态）、没有本能反应（安全护栏），什么都做不了。

## 三个核心理念

### 1. Agent Loop — 让模型自己开车

```
普通 API 调用：人问 → 模型答 → 结束
Agent Harness：人给目标 → 模型自己决定做什么 → 调工具 → 看结果 → 再决定 → 循环直到完成
```

关键区别：**控制权在模型手里**，不是调用方。

### 2. 决定可靠性的是 Harness，不是模型

LangChain 不换模型，只改 Harness，排名从 #30 到 #5。

这意味着：工具设计好不好、上下文给得对不对、安全检查够不够、死循环能不能检测——这些"基础设施"比模型本身更决定结果。

### 3. 把失败变成永久修复，而不是重试

Agent 犯了错 → 不是重跑一次，而是改规则/加检查/更新指令，让这类错误不再发生。Harness 是不断进化的。

## 最快理解方式

**Claude Code 本身就是一个 Agent Harness 的完整实现：**

- **Agent Loop** — 模型自己决定读什么文件、改什么代码
- **工具层** — Read/Write/Edit/Bash/Grep...
- **安全层** — 危险操作问你确认
- **上下文管理** — 对话太长自动压缩
- **记忆** — MEMORY.md 跨会话记住你

回顾之前的任务——读目录、搜资料、创建 17 个文件、自己决定结构和内容。**这整个过程就是 Harness 在工作。**
